说明：
1、对code_message.json的说明：

    作用：断言，主要是断言预期接口和实际结果是否相符

   维护了以下内容：
       {
        "api3/getbanneradvertver2":[
            {"1006":"token error"},
            {"1007":"用户名错误"},
            {"1008":"密码错误"}
        ],
        "api3/beta4":[
            {"1006":"登陆成功"},
            {"1006":"用户名错误"},
            {"1006":"密码错误"}
            ]
        }
   1006是接口返回的错误码，token error是接口返回的错误信息，也就是维护了：错误码和错误信息提示的对应关系，
   断言的时候，获取接口返回的错误信息描述，和code_message已经维护好的错误信息描述进行对比，看用例是否执行通过 ，run_main()中用到


2、对result.json的说明：

    作用：断言，主要是断言预期接口的响应报文和实际结果的报文的结构体是否相符

    result.json主要是维护接口响应的全部报文，包含sucess和error的部分，在 rum_main()中的代码:

            if except_method == "json":
                ..... ......

    通过传入接口的实际响应和获取result.json中的报文进行两个字典的对比


3、cookie.json
作用：存放cookie的文件，以键值对的方式存放，键是代表一个标识，比如使用app键存放慕课网的cookie

4、header.json
存放请求头额文件
